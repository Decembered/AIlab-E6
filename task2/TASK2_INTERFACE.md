# Task2 Hand Reconstruction Interface

This folder contains the Task2 hand reconstruction and trajectory recovery pipeline for Video2Motion2Action.

Committed contents include code, environment files, documentation, reports, and a curated GitHub-safe visualization set under `media/`. Raw challenge videos, extracted frames, generated full outputs, logs, third-party source trees, and model weights are intentionally excluded.

## Folder Layout

- `scripts/`: runnable pipeline scripts.
- `src/`: reusable utilities and shared hand schema constants.
- `env/`: conda and pip environment files.
- `docs/`: SAM2 and HaMeR extension plans.
- `reports/`: method notes, delivery index, quality reports, and result summaries.
- `models/`: placeholder README files describing where to place MANO, MediaPipe, SAM2, and HaMeR files.
- `media/`: selected action visualization videos small enough for normal GitHub storage.

## Data To Fill In

Create these local folders after cloning, if needed:

```text
task2/data/raw/          # optional raw videos or copied local inputs
task2/data/frames/       # extracted frames generated by 01_extract_frames.py
task2/data/processed/    # camera calibration or intermediate metadata
task2/outputs/           # generated trajectories, masks, overlays, videos, figures
task2/logs/              # run logs
task2/third_party/       # optional local clones of HaMeR, SAM2, etc.
```

Required model/data files are not committed. Place them locally according to the README files under `task2/models/`.

## Main Output Contract

Task2 exports hand trajectories as:

```text
task2/outputs/trajectories/hand_traj.npz
```

Core fields used by Task1 and Task3:

- `sequence_id`, `fps`, `frame_ids`, `timestamps`
- `keypoints2d`, `keypoints2d_score`
- `keypoints3d`, `keypoints3d_score`
- `wrist_pos`, `wrist_rot`, `palm_pos`, `palm_rot`
- `T_world_wrist`, `T_world_palm`
- `fingertips3d`, `fingertips3d_palm`
- `retarget_landmark_names`, `retarget_keypoints3d`, `retarget_keypoints3d_palm`, `retarget_weights`
- `valid`, `quality_score`, `interpolated_flag`, `failure_reason`
- validity flags such as `metric_3d_valid`, `world_alignment_valid`, `camera_calib_valid`, `contact_valid`, `phase_valid`

See `reports/task2_delivery_index.md` and `scripts/08_validate_hand_traj.py` for the detailed schema.

## Typical Run

From the repository root:

```bash
conda env create -f task2/env/environment_task2.yml
conda activate v2m2a-task2
bash task2/scripts/00_check_env.sh
```

Run a single-view baseline after providing the video path:

```bash
bash task2/scripts/20_run_single_view_baseline.sh \
  --video /path/to/video.mkv \
  --sequence_id your_sequence_id \
  --view_id camera_side_2 \
  --fps 15
```

Validate the exported interface:

```bash
python task2/scripts/08_validate_hand_traj.py \
  --traj_npz task2/outputs/trajectories/hand_traj.npz \
  --out_report task2/reports/hand_traj_validation.md
```

Optional multi-view triangulation entry point:

```bash
python task2/scripts/15_triangulate_multiview_hand_traj.py --help
```

## Notes For Task1 And Task3

- Task1 can provide local input videos or extracted frames; this repository does not commit challenge raw data.
- Task3 should consume `task2/outputs/trajectories/hand_traj.npz`, multi-view outputs, or by-view archived trajectories after Task2 has run locally.
- Current baseline is MediaPipe non-metric 3D, with optional multi-view triangulation support when local camera/view inputs are available.
- MANO/HaMeR/SAM2 are documented extension points, not required for the committed baseline to run.
